NetAtlas Scout

A local network discovery sensor that observes Layer 2 presence,
maps IP-to-MAC relationships, detects changes, and reports device
observations to NetAtlas.

1. Passive ARP observation
   - Läs ARP-tabellen
   - Se vilka IP/MAC-par som redan är kända
   - Upptäck nya enheter utan aktiv scanning

2. Active ARP scan
   - Skanna ett subnet, t.ex. 192.168.1.0/24
   - Fråga: "vem har 192.168.1.42?"
   - Samla IP, MAC och svarstid

3. Device presence
   - First seen
   - Last seen
   - Seen count
   - Currently active / inactive

4. Change detection
   - Ny MAC på känd IP
   - Ny IP på känd MAC
   - Duplicate IP
   - Gateway MAC changed
   - Device disappeared
   - Device returned

5. Vendor lookup
   - MAC OUI lookup
   - Apple, Raspberry Pi, Zebra, Intel, etc.

6. Confidence model
   - ARP seen = stark lokal bekräftelse
   - ARP + DHCP hostname = starkare
   - ARP + DNS activity = ännu starkare

7. Event generation
   - device_seen
   - device_new
   - device_lost
   - ip_changed
   - mac_changed
   - duplicate_ip
   - gateway_changed

ARP-0.1 Concept & Scope
=============================================================================================================================================================================================

Goal:
Define the role of the ARP sensor as the local Layer 2 discovery component in NetAtlas.

Scope:
- Name the sensor
- Define passive vs active discovery
- Define ARP observations
- Define what the sensor should and should not decide
- Define initial event types

Deliverables:
- Sensor name
- Sensor description
- Observation model draft
- Event list draft

Status:

Implemented.

Implementation notes:
- Sensor name is `arpscout`.
- Observation model is `internal/arpscout.Observation`.
- Initial event constants are defined for device_seen, device_new, device_lost, ip_changed,
  mac_changed, duplicate_ip, and gateway_changed.
- `arpscout info` prints sensor scope and event types as JSON.

ARP-0.2 Passive ARP Reader
=============================================================================================================================================================================================

Goal:
Read the local ARP/neighbour table and emit basic observations.

Scope:
- Read Linux neighbour table using `ip neigh`
- Parse IP, MAC, interface and state
- Support one or more interfaces
- Ignore incomplete entries unless configured otherwise
- Emit normalized ARP observations

Deliverables:
- CLI command
- Parser
- Basic observation output as JSON
- Unit tests with sample `ip neigh` output

Status:

Implemented.

Implementation notes:
- Added `arpscout passive`.
- Reads Linux neighbour table using `ip neigh show`.
- Supports `-iface` for one or more interfaces.
- Ignores INCOMPLETE entries by default and supports `-include-incomplete`.
- Emits normalized observations as JSON.
- Parser tests cover IPv4, IPv6, interface filtering, and incomplete entries.

=============================================================================================================================================================================================
ARP-0.4 Active ARP Scan
=============================================================================================================================================================================================

Goal:
Actively discover devices on a configured subnet.

Scope:
- Configure subnet CIDR
- Send ARP requests
- Collect replies
- Rate limit scan
- Avoid aggressive behaviour
- Mark source as active_scan
- Require explicit subnet input
- Support dry-run scan planning

Deliverables:
- `arpscout scan`
- CIDR parser
- Active scan result output as JSON
- Scan safety limits
- Unit tests

Status:

Implemented.

Implementation notes:
- Added `arpscout scan`.
- Requires explicit `-cidr` input and only supports IPv4 ARP ranges.
- Supports `-dry-run` to plan targets without sending ARP requests.
- Enforces a configurable safety limit with `-max-hosts`, defaulting to 256 targets.
- Supports `-iface`, `-timeout`, and `-interval` for controlled active scanning.
- Active observations are marked with source `active_scan`.
- The implementation shells out to `arping` for packet sending and keeps CIDR planning/parser logic unit tested.

=============================================================================================================================================================================================
ARP-0.4.1 Scan Target Discovery & CLI UX
=============================================================================================================================================================================================

Goal:
Make active scanning easier and safer by discovering reasonable scan targets automatically and improving scan command feedback.

Scope:
- Allow `arpscout scan` without explicit CIDR.
- Auto-detect local interface networks.
- Classify scan targets before scanning.
- Scan normal physical LAN interfaces by default.
- Skip loopback, Docker bridges, virtual bridges, VPN and container networks by default.
- Allow explicit CIDR scan with `arpscout scan <cidr>`.
- Allow explicit interface scan with `arpscout scan -iface enp3s0`.
- Allow all discovered networks with `arpscout scan -all`.
- Show clear usage when no valid scan target exists.
- Show scan plan before active scan starts.
- Improve missing CIDR error message.

Deliverables:
- Scan target discovery.
- Interface/network classifier reuse.
- Default scan target selection.
- `-iface` scan target support.
- `-all` scan mode.
- Human-readable scan plan.
- Better CLI help and error messages.
- Unit tests for physical, loopback, Docker, bridge and VPN interfaces.

Status:

Implemented.

Implementation notes:
- Added scan target discovery from local interface IPv4 networks.
- `arpscout scan` can run without CIDR and selects the first normal physical LAN target.
- Explicit CIDR is supported with both `-cidr 192.168.1.0/24` and `arpscout scan 192.168.1.0/24`.
- `-iface` filters discovered targets or binds an explicit CIDR scan to an interface.
- `-all` includes discovered networks that are normally skipped, while existing safety limits still apply.
- Loopback, Docker-style bridges, virtual bridges, VPN and container networks are skipped by default.
- `-format text` prints a human-readable scan plan and result summary.
- Unit tests cover physical, loopback, Docker bridge, VPN, `-all`, interface filtering and positional CIDR.

ARP-0.4.2 Scan Output Model
=============================================================================================================================================================================================

Goal:
Make active scan output useful for both humans and machines without flooding normal output with uninteresting planned target lists.

Scope:
- Separate scan plan from scan result.
- Do not include full planned target lists in normal output.
- Report scan summary instead of every possible target.
- Include discovered observations in normal scan output.
- Include target count, first target and last target in dry-run output.
- Include full planned target list only in debug output.
- Add reply count and scan duration.
- Add network type/classification to scan result.
- Keep JSON output stable and predictable.

Deliverables:
- ScanPlan model.
- ScanResult model.
- Human-readable scan summary.
- JSON scan result output.
- Dry-run summary output.
- Debug-only planned target output.
- Unit tests for normal, dry-run and debug output.

Status:

Implemented.

Implementation notes:
- Added `ScanPlan` and `ScanStatistics` to active scan results.
- Normal JSON output reports scan plan and statistics instead of a full planned target list.
- Dry-run output includes target count, first target and last target.
- Full planned target output is only included with `-debug`.
- Text output prints a concise scan summary with reply count, scanned count and duration.
- Scan result includes address and network classification through the plan model.

- Network classification module.

Status:

Implemented.

=============================================================================================================================================================================================
ARP-0.6 Change Detection
=============================================================================================================================================================================================

Goal:
Detect meaningful changes between observation batches without turning the sensor into the system of record.

Scope:
- Compare current batch with previous in-memory state
- New device seen in current run
- Device missing from current run
- MAC changed for IP
- IP changed for MAC
- Duplicate IP in current batch
- Gateway MAC changed
- Emit events as observations, not final truth

Deliverables:
- Event generator
- Event constants
- Human-readable event output
- JSON event output
- Unit tests

Status:

Implemented.

Implementation notes:
- Added an in-memory change detector for comparing observation batches.
- Added `arpscout changes` for comparing two observation JSON files.
- Emits JSON by default and supports human-readable text with `-format text`.
- Detects new devices, missing devices, MAC changes for IPs, IP changes for MACs,
  duplicate IPs in the current batch, and gateway MAC changes with `-gateway`.
- Does not write local state or use a database; daemon mode can reuse the same in-memory detector later.
- Unit tests cover event generation, in-memory detector behaviour, and CLI text output.
